# 服务监控

服务监控模块主要是用户检查服务运行状态，并且支持通过UI界面重启服务。

# 添加服务监控

1、docker名称：自己命名的服务名称 2、环境名称：该服务所在的环境名称 3、检查CMD：用于检查服务是否连通的curl语句 4、重启CMD：用于重启服务的命令 5、停止CMD：用于停止服务的命令
